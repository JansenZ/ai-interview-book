## 📖 核心术语表

### 1. AI / ML / DL / LLM 的关系

```
AI (人工智能)
├── ML (机器学习)
│   ├── DL (深度学习)
│   │   └── LLM (大语言模型)
│   ├── 传统机器学习 (SVM、决策树等)
│   └── 强化学习
└── 规则系统 (非学习)
```

**一句话解释**：
- **AI**：让机器模拟人类智能的总称
- **ML**：让机器从数据中学习，而不是硬编码规则
- **DL**：用神经网络（多层）的机器学习
- **LLM**：用 Transformer 架构的超大规模语言模型

### 2. 模型相关术语

| 术语 | 解释 | 前端类比 |
|------|------|----------|
| **模型 (Model)** | 训练好的 AI 程序 | 一个封装好的 npm 包 |
| **训练 (Training)** | 用数据教模型学习 | 写代码 + 测试迭代 |
| **推理 (Inference)** | 用模型做预测 | 调用 API 获取结果 |

| ** epoch** | 完整遍历一遍训练数据 | 一轮完整的测试 |
| **Batch Size** | 一次训练多少样本 | 并发请求数 |
| **Learning Rate** | 学习速度 | 迭代更新的步长 |

### 3. 大模型专用术语

| 术语 | 解释 | 面试考点 |
|------|------|----------|
| **Token** | 文本的最小单位 | 1 个中文≈1.5 个 token，1 个英文单词≈1.3 个 token |
| **Context Window** | 模型能处理的上下文长度 | GPT-4: 128K, Claude: 200K |
| **Temperature** | 输出随机性 (0-1) | 0=确定，1=随机 |
| **System Prompt** | 系统级指令 | 设定 AI 角色 |
| **User Prompt** | 用户输入 | 具体问题 |
| **Completion** | 模型生成的内容 | API 返回结果 |
| **Fine-tuning** | 在基础模型上继续训练 | 让模型更专业 |
| **Prompt Engineering** | 设计更好的提示词 | 前端必会 |

**Token 计算示例**：
```javascript
// "你好，世界" ≈ 6-8 个 token
// "Hello, World" ≈ 4 个 token
// 1000 个中文字 ≈ 1500 个 token
```

### 6. 评估指标

| 指标 | 解释 | 面试考频 |
|------|------|----------|
| **Accuracy** | 准确率 | ⭐ |
| **Precision** | 精确率 | ⭐⭐ |
| **Recall** | 召回率 | ⭐⭐ |
| **F1 Score** | 精确率和召回率的调和平均 | ⭐⭐⭐ |
| **Perplexity** | 困惑度（越低越好） | ⭐ |
| **BLEU** | 翻译质量评估 | ⭐ |
| **ROUGE** | 文本摘要评估 | ⭐ |

**精确率 vs 召回率（面试必考）**：
```
精确率：预测为正的里面，多少是真的正
召回率：真的正里面，多少被预测出来了

例子：疫情检测
- 精确率高 = 检测为阳性的人，真的都是阳性
- 召回率高 = 真的阳性的人，基本都被检测出来了
```

## 💡 快速记忆技巧

### 类比法

| AI 概念 | 前端类比 |
|--------|----------|
| 模型训练 | 写代码 + 测试迭代 |
| 模型推理 | 调用 API |
| Prompt | 函数参数 |
| Token | 请求/响应大小 |
| Context Window | 内存限制 |
| Embedding | 数据序列化 |
| Vector DB | 索引数据库 |
| RAG | 搜索引擎 + API |

### 公式法

```
AI 应用 = Prompt + 模型 API + 业务逻辑

RAG 应用 = 用户问题 → 检索相关文档 → 拼接到 Prompt → 调用模型 → 返回答案

Function Calling = 用户问题 → 模型决定调用哪个函数 → 执行函数 → 结果返回给模型 → 最终答案
```

## 📝 自测题

**1. 解释 AI、ML、DL、LLM 的区别和联系**

<details>
<summary>点击查看答案</summary>

AI 是总称，ML 是 AI 的子集（从数据学习），DL 是 ML 的子集（用神经网络），LLM 是 DL 的一种（大语言模型）。

关系：AI > ML > DL > LLM
</details>

**2. Token 是什么？1000 个中文字大约多少 token？**

<details>
<summary>点击查看答案</summary>

Token 是文本的最小处理单位。1000 个中文字 ≈ 1500 个 token。
</details>

**3. 什么是 RAG？为什么前端需要知道？**

<details>
<summary>点击查看答案</summary>

RAG = 检索增强生成。先检索相关文档，再让 AI 基于文档回答。前端需要知道因为这是最常见的 AI 应用场景。
</details>

**4. Temperature 参数有什么用？面试时怎么说？**

<details>
<summary>点击查看答案</summary>

Temperature 控制输出随机性（0-1）。0=确定性好，适合事实性问题；1=随机性强，适合创意场景。面试时可以说："根据场景选择合适的 temperature，客服用 0.3，创意写作可以用 0.8"。
</details>
