## 🏛️ 1950s-1960s: AI 诞生

### 关键事件

| 年份 | 事件 | 意义 |
|------|------|------|
| **1950** | 图灵测试提出 | AI 的哲学基础 |
| **1956** | 达特茅斯会议 | "人工智能"一词诞生 |
| **1957** | 感知机（Perceptron） | 第一个神经网络模型 |
| **1966** | ELIZA 聊天机器人 | 第一个对话系统 |

### 图灵测试（面试必考）

**定义**：如果一台机器和人类对话，人类无法区分它是机器还是人，就说它有智能。

```
人类评判员
    │
    ├── 和人类 A 聊天
    │
    └── 和机器 B 聊天
    
如果评判员分不清 A 和 B 谁是机器 → 机器通过图灵测试
```

**面试考点**：
> **Q: 什么是图灵测试？**
> 
> **A**: 
> 图灵 1950 年提出的测试，判断机器是否有智能。
> 方法：人类评判员同时和一个人、一台机器聊天，
> 如果评判员分不清谁是机器，就说机器通过了测试。
> 
> 意义：为 AI 设定了一个可操作的目标。
> 局限：只测试对话能力，不测试其他智能。

### ELIZA 聊天机器人

**原理**：模式匹配 + 规则回复（不是 AI！）

```
用户：我心情不好
ELIZA: 为什么你心情不好？（规则：把"我 X"变成"为什么你 X"）

用户：我爸爸打我
ELIZA: 你家里还有谁打你？（规则：聚焦家庭关系）
```

**启示**：即使没有真正的理解，也能做出看似智能的行为。

## ❄️ 1970s-1980s: AI 寒冬

### 为什么叫"寒冬"？

**原因**：期望太高，技术达不到 → 资金断崖

```
1960s：AI 将很快超越人类！（过度乐观）
    ↓
1970s：发现根本做不到（现实打击）
    ↓
政府和企业停止投资（寒冬来临）
    ↓
AI 研究者找不到工作（人才流失）
```

### 教训（面试可以聊）

> **启示**：
> 1. 技术成熟需要时间，不能急于求成
> 2. 期望管理很重要
> 3. 现在的 AI 热潮也要理性看待

## 🧠 1980s-1990s: 专家系统时代

### 专家系统（Expert System）

**核心思想**：把人类专家的知识写成规则

```
// 医疗诊断专家系统
if (症状 == "发烧" AND 咳嗽 == "有") {
  可能诊断 = "感冒";
}

if (症状 == "胸痛" AND 年龄 > 50) {
  建议 = "立即就医";
}
```

**优点**：
- 可解释（知道为什么做出这个判断）
- 特定领域效果好

**缺点**：
- 规则要人工写，成本高
- 无法处理未知情况
- 难以扩展

**衰落原因**：
- 规则太多，维护困难
- 无法学习新知识
- 机器学习兴起

## 🎮 1997: DeepBlue 击败人类棋王

### 事件

**IBM DeepBlue** vs **国际象棋世界冠军 Kasparov**

结果：DeepBlue 3.5 : 2.5 获胜

### 意义

- 第一次计算机在复杂智力游戏中击败人类
- 证明计算机在特定领域可以超越人类
- 但 DeepBlue 是暴力搜索，不是真正的 AI

**面试考点**：
> **Q: DeepBlue 是怎么赢的？**
> 
> **A**: 
> 不是靠机器学习，而是暴力搜索 + 评估函数。
> 每秒计算 2 亿步棋，往后推 12 步，
> 用评估函数判断局面好坏。
> 
> 意义：证明计算机在特定领域可以超越人类。
> 局限：只能下棋，不能做其他事（窄 AI）。

## 🖼️ 2012: 深度学习崛起

### ImageNet 竞赛

**事件**：AlexNet 以压倒性优势赢得图像识别比赛

```
2011 年冠军：错误率 25.7%（传统方法）
2012 年冠军：错误率 15.3%（AlexNet，深度学习）
2013 年：错误率 11.7%
2015 年：错误率 3.6%（超越人类）
```

### 为什么重要？

1. **证明深度学习有效**：之前大家还怀疑
2. **GPU 加速**：发现 GPU 适合训练神经网络
3. **大数据**：ImageNet 有 1400 万标注图片
4. **开启 AI 热潮**：资本和人才大量涌入

### AlexNet 架构（了解即可）

```
输入图片 → 卷积层 1 → 池化 → 卷积层 2 → 池化 → 全连接 → 输出
         (11x11 卷积)        (5x5 卷积)          (4096 神经元)
```

**面试考点**：
> **Q: 2012 年 AI 领域发生了什么大事？**
> 
> **A**: 
> AlexNet 赢得 ImageNet 图像识别竞赛，
> 错误率从 25% 降到 15%，碾压传统方法。
> 
> 意义：
> 1. 证明深度学习有效
> 2. GPU 加速成为标准
> 3. 开启了新一轮 AI 热潮
> 
> 从那以后，深度学习成为 AI 的主流方法。
```

## 🤖 2017: Transformer 诞生

### 论文：《Attention Is All You Need》

**作者**：Google Brain 团队

**核心思想**：只用 Attention 机制，不用 RNN/CNN

```
传统方法（RNN）：
输入 → RNN 单元 1 → RNN 单元 2 → ... → RNN 单元 N → 输出
       （顺序处理，慢）

Transformer：
输入 → Self-Attention → 前馈网络 → 输出
      （并行处理，快）
```

### 为什么重要？

1. **并行计算**：比 RNN 快几十倍
2. **长距离依赖**：能处理长文本
3. **可扩展**：可以做得很大
4. **通用架构**：文本、图片、音频都能用

### 影响

```
2018: BERT（Google）- 双向 Transformer
2019: GPT-2（OpenAI）- 15 亿参数
2020: GPT-3（OpenAI）- 1750 亿参数
2022: ChatGPT - 引爆全球
2023: GPT-4 - 多模态
```

**面试考点**：
> **Q: Transformer 为什么重要？**
> 
> **A**: 
> 1. 只用 Attention 机制，架构简洁
> 2. 并行计算，训练速度快
> 3. 能处理长距离依赖
> 4. 成为大模型的基础架构
> 
> 现在所有大模型（GPT、Claude、通义千问）都基于 Transformer。
```

## 🚀 2020-2022: 大模型爆发

### GPT 系列发展

| 模型 | 时间 | 参数 | 意义 |
|------|------|------|------|
| **GPT** | 2018.06 | 1.17 亿 | 证明 Transformer 能做生成 |
| **GPT-2** | 2019.02 | 15 亿 | 能写连贯文章，一度不公开 |
| **GPT-3** | 2020.06 | 1750 亿 | 零样本学习，能力涌现 |
| **ChatGPT** | 2022.11 | 未知 | 对话优化，引爆全球 |
| **GPT-4** | 2023.03 | 未知 | 多模态，能力更强 |

### GPT-3 的"涌现能力"

**定义**：模型大到一定程度，突然学会了没训练过的能力

```
GPT-2：不会做算术
GPT-3：会做简单算术（没人教）

GPT-2：不会翻译（没训练过）
GPT-3：会翻译多种语言（零样本）

GPT-2：代码能力弱
GPT-3：能写简单代码
```

**启示**：规模（数据 + 参数）带来质变

## 💥 2022.11: ChatGPT 引爆全球

### 事件

OpenAI 发布 ChatGPT，5 天用户破 100 万

```
ChatGPT 用户增长：
5 天：100 万
2 个月：1 亿（史上增长最快的应用）
```

### 为什么爆火？

1. **对话流畅**：像真人一样聊天
2. **什么都会**：写代码、写文章、做题...
3. **免费使用**：门槛低
4. **社交传播**：各种截图刷屏

### 影响

```
科技行业：
- 所有大厂都在做大模型
- AI 创业公司估值飙升
- 传统软件都在加 AI 功能

社会影响：
- 讨论 AI 会不会取代人类
- 教育界担心作弊
- 政府开始监管 AI
```

**面试考点**：
> **Q: 你怎么看 ChatGPT 的影响？**
> 
> **A**: 
> ChatGPT 是 AI 发展的里程碑事件。
> 
> 技术层面：证明大模型在对话场景的实用性。
> 商业层面：开启了 AI 应用的商业化浪潮。
> 社会层面：引发对 AI 影响的广泛讨论。
> 
> 对前端开发的影响：
> - 几乎所有产品都要加 AI 功能
> - 前端需要学习如何集成 AI
> - 出现了新的交互模式（对话式 UI）
```

## 🎨 2023: 多模态元年

### 什么是多模态？

**定义**：能同时处理多种类型的数据（文本、图片、音频、视频）

```
GPT-4：能看懂图片，回答相关问题
输入：[图片] + "图片里有什么？"
输出："图片里有一只猫在沙发上睡觉"

Claude 3：能读 PDF、图表、截图
输入：[财务报表截图] + "分析这家公司"
输出：财务分析报告
```

### 代表性模型

| 模型 | 公司 | 多模态能力 |
|------|------|-----------|
| **GPT-4V** | OpenAI | 图片理解 |
| **Claude 3** | Anthropic | 图片、图表、PDF |
| **Gemini** | Google | 图片、音频、视频 |
| **通义千问 VL** | 阿里 | 图片理解 |

### 应用场景

- **客服**：用户发截图，AI 识别问题
- **教育**：拍题搜答案
- **办公**：分析图表、PPT
- **电商**：以图搜图

## 🤖 2024: Agent 元年

### 什么是 AI Agent？

**定义**：能自主规划、使用工具、完成复杂任务的 AI

```
传统 AI：
用户：写一个排序函数
AI：[直接输出代码]

AI Agent：
用户：帮我分析这个数据集
AI：
  1. 先读取数据文件
  2. 做数据清洗
  3. 做统计分析
  4. 生成可视化图表
  5. 写分析报告
```

### Agent 的核心能力

1. **规划（Planning）**：拆解复杂任务
2. **工具使用（Tool Use）**：调用 API、执行代码
3. **记忆（Memory）**：记住之前的操作
4. **反思（Reflection）**：检查错误并修正

### 代表性产品

| 产品 | 公司 | 特点 |
|------|------|------|
| **Devin** | Cognition | AI 软件工程师 |
| **AutoGPT** | 开源 | 自主任务执行 |
| **LangChain Agent** | LangChain | 框架级支持 |

### 对前端的影响

```
1. 前端也要支持 Agent 交互
   - 不只是对话，还要展示 Agent 的操作过程
   
2. 前端本身可以被 Agent 操作
   - AI 可以自动点击、填写表单
   
3. 新的开发模式
   - 用自然语言描述需求，AI 生成前端代码
```

## 📊 AI 发展规律总结

### 1. 算力驱动

```
1950s: CPU 还没诞生 → AI 只能做简单推理
1980s: 个人电脑出现 → 专家系统可行
2012: GPU 加速 → 深度学习爆发
2020: 专用 AI 芯片 → 大模型可行
```

### 2. 数据驱动

```
早期：人工标注数据，量少
2012: ImageNet 1400 万图片
2020: 互联网全部文本
2023: 多模态数据（图片 + 文本）
```

### 3. 算法演进

```
1950s: 规则系统
1980s: 专家系统
1990s: 统计机器学习
2012: 深度学习
2017: Transformer
2020: 大语言模型
```

### 4. 能力边界扩展

```
1950s: 逻辑推理
1980s: 特定领域知识
1997: 国际象棋
2012: 图像识别
2017: 文本生成
2022: 对话、代码
2023: 多模态
2024: 自主 Agent
```

## 🎯 面试高频问题

### Q1: AI 发展经历了几个阶段？

**参考回答**：
```
大致可以分为几个阶段：

1. 1950s-1960s：诞生期
   - 图灵测试、感知机、ELIZA
   - 过度乐观，埋下寒冬种子

2. 1970s-1980s：第一次寒冬 + 专家系统
   - 期望落空，资金断裂
   - 专家系统兴起，但难以扩展

3. 1990s-2000s：机器学习兴起
   - 统计方法取代规则
   - DeepBlue 击败棋王

4. 2012-2017：深度学习崛起
   - AlexNet 赢得 ImageNet
   - Transformer 诞生

5. 2018-2022：大模型时代
   - GPT 系列不断突破
   - ChatGPT 引爆全球

6. 2023-至今：多模态 + Agent
   - GPT-4 多模态
   - AI Agent 自主任务
```

**参考回答**：

我认为这次和以前不同，有实质性突破：
