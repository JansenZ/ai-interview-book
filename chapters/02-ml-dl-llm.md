## 1️⃣ 机器学习 (Machine Learning)

### 核心思想

**传统编程**：规则 + 数据 → 答案
**机器学习**：数据 + 答案 → 规则

```javascript
// 传统编程：判断邮件是否是垃圾邮件
function isSpam(email) {
  if (email.contains('中奖')) return true;
  if (email.contains('免费')) return true;
  if (email.from === 'unknown') return true;
  return false;
}

// 机器学习：让模型自己学习规则
const model = train(emailDataset); // 数据包含邮件和是否是垃圾的标签
const result = model.predict(newEmail); // 模型自动判断
```

### 典型应用场景

| 场景 | 传统 ML 算法 | 前端相关度 |
|------|------------|-----------|
| 垃圾邮件检测 | 逻辑回归、SVM | ⭐ |
| 用户分类 | K-Means 聚类 | ⭐⭐ |
| 推荐系统 | 协同过滤 | ⭐⭐⭐ |
| 异常检测 | 孤立森林 | ⭐⭐ |
| 销量预测 | 线性回归 | ⭐⭐ |

### 前端开发需要知道吗？

**知道概念就行**，实际工作中很少直接用传统 ML 算法。但面试可能会问：

> **面试问题**：机器学习和传统编程有什么区别？
>
> **参考回答**：
> 传统编程是人工写规则，机器执行；机器学习是从数据中自动学习规则。
> 比如垃圾邮件检测，传统方式要人工写规则（包含"中奖"就是垃圾邮件），
> 机器学习是给模型大量标注数据，让它自己学习规律。
> 机器学习的优势是能处理复杂模式，不需要人工枚举所有规则。

## 2️⃣ 深度学习 (Deep Learning)

### 核心思想

深度学习 = 机器学习 + 神经网络（多层）

```
输入层 → 隐藏层 1 → 隐藏层 2 → ... → 隐藏层 N → 输出层
         (神经元)    (神经元)          (神经元)
```

**为什么叫"深度"**：因为有很多层（deep = 多层）

### 和传统 ML 的区别

| 对比项 | 传统 ML | 深度学习 |
|--------|---------|----------|
| 特征工程 | 需要人工设计特征 | 自动学习特征 |
| 数据需求 | 小数据也能用 | 需要大量数据 |
| 计算资源 | CPU 就够了 | 需要 GPU |
| 可解释性 | 较好理解 | 黑盒，难理解 |
| 性能 | 小数据好 | 大数据碾压 |

**特征工程举例**：
```javascript
// 传统 ML：需要人工设计特征
const features = {
  emailLength: email.length,
  hasMoneySymbol: email.includes('¥'),
  senderDomain: extractDomain(email.from),
  // ... 需要领域专家设计
};

// 深度学习：原始文本直接输入，自动学习特征
const input = email.rawText;
const result = deepModel.predict(input);
// 模型自己学习哪些模式重要
```

### 常见深度学习模型

| 模型 | 用途 | 前端应用场景 |
|------|------|-------------|
| **CNN** | 图像识别 | 图片分类、目标检测 |
| **RNN/LSTM** | 序列处理 | 文本生成、语音识别 |
| **Transformer** | 序列处理 | 翻译、文本生成 |
| **GAN** | 图像生成 | AI 绘画、风格迁移 |

### 前端相关度

**浏览器端深度学习**：
- TensorFlow.js - 在浏览器运行深度学习模型
- ONNX Runtime Web - 跨平台推理

```javascript
// 示例：用 TensorFlow.js 做图片分类
const model = await tf.loadLayersModel('model.json');
const prediction = model.predict(imageTensor);
```

## 3️⃣ 大语言模型 (LLM)

### 核心定义

LLM = 基于 Transformer 架构 + 超大规模参数 + 海量文本训练

**关键特点**：
1. **参数多**：GPT-3 有 1750 亿参数
2. **数据多**：训练数据来自整个互联网
3. **通用性强**：什么任务都能做一点

### LLM vs 传统深度学习

| 对比项 | 传统 DL | LLM |
|--------|---------|-----|
| 训练数据 | 特定领域数据 | 互联网全部文本 |
| 模型规模 | 几百万到几亿参数 | 几百亿到万亿参数 |
| 训练成本 | 几万到几十万 | 几百万到上亿 |
| 使用方式 | 针对特定任务 | 通用，prompt 控制 |
| 泛化能力 | 只能做训练的任务 | 零样本学习新任务 |

**零样本学习 (Zero-shot)**：
```
传统 DL：要识别新类别，需要重新收集数据训练
LLM：直接问"这是猫还是狗？"，没见过也能答
```

### LLM 的能力层次

```
Level 1: 文本生成（续写、补全）
Level 2: 问答（基于训练数据）
Level 3: 推理（逻辑、数学）
Level 4: 工具使用（调用 API、执行代码）
Level 5: 自主规划（Agent，多步骤任务）
```

### 前端开发需要掌握什么？

**⭐⭐⭐ 必须会**：
1. 调用 LLM API（OpenAI、Claude、通义千问）
2. Prompt Engineering（提示词工程）
3. RAG（检索增强生成）
4. Function Calling（让模型调用函数）

**⭐⭐ 了解**：
1. Fine-tuning（微调）
2. Embedding（文本向量化）

**⭐ 知道就行**：
1. Transformer 架构细节
2. 训练过程

## 🎯 面试高频问题

### Q1: 机器学习、深度学习、大模型的区别？

**参考回答**：
```
从属关系：AI > ML > DL > LLM

机器学习是从数据中学习规则，而不是人工写规则。
深度学习是机器学习的一种，用多层神经网络，能自动学习特征。
大模型是深度学习的一种，基于 Transformer 架构，参数规模超大（几百亿+），
训练数据是互联网文本，有很强的通用能力。

关键区别：
1. 数据量：ML 小数据可用，DL 需要大数据，LLM 需要海量数据
2. 特征工程：ML 需要人工设计，DL 自动学习
3. 通用性：ML/DL 针对特定任务，LLM 通用
4. 使用方式：ML/DL 训练后部署，LLM 用 prompt 控制
```

### Q2: 为什么大模型能零样本学习？

**参考回答**：
```
大模型在训练时见过互联网上几乎所有类型的文本，
包括问答、翻译、代码、数学题等各种任务的示例。

虽然训练时没有明确标注"这是 XX 任务"，
但模型从海量数据中学会了各种任务的 pattern。

当遇到新任务时，prompt 的形式会激活模型中相关的 pattern，
所以即使没专门训练过，也能完成新任务。

这就是"涌现能力"——规模大到一定程度，突然就会了很多东西。
```

### Q3: 前端开发为什么要学 AI？

**参考回答**：
```
1. 产品需求：现在几乎所有产品都要加 AI 功能
   - 智能客服、内容生成、个性化推荐
   
2. 技术趋势：AI 正在改变前端开发方式
   - Copilot 写代码、AI 生成 UI、自动测试
   
3. 竞争力：懂 AI 的前端更稀缺
   - 会用 AI 的前端 > 只会传统前端
   - 能设计 AI 应用架构的前端 > 只会调 API
   
4. 实际场景：前端是 AI 应用的最后一公里
   - 再好的 AI 模型，也需要前端集成到产品中
   - 用户体验、性能优化、成本控制都需要前端
```
